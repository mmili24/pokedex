Descripció del projecte

Has de construir una PokéDex web de múltiples pàgines que permeti cercar Pokémon, consultar-ne el detall i disputar batalles simulades. L'aplicació consumirà l'API pública PokéAPI (https://pokeapi.co/api/v2) per obtenir totes les dades en temps real.

Se t'entrega l'estructura HTML de l'index, que no pots modificar, i els fulls d'estil CSS de totes les pàgines. A banda el fitxer per consumir l'API amb el nom de les funcions que no podras modificar. La teva tasca és implementar tot el JavaScript que doni vida a l'aplicació, incloent-hi la lògica de batalla, i els estils CSS.

API de referència Explora l'API des del navegador: https://pokeapi.co/api/v2/pokemon/25 per a dades d'un Pokémon, o https://pokeapi.co/api/v2/pokemon?limit=151 per al llistat. Llegeix la documentació oficial a pokeapi.co/docs/v2 per entendre tots els endpoints disponibles.

Pàgines i funcionalitats

🏠 index.html — Buscador Obligatori
  • Carrega i mostra els primers 151 Pokémon en graella
  • Cerca en temps real per nom o número
  • Cada targeta navega a la pàgina de detall
  • Mostra tipus amb colors corresponents
🔍 details.html — Detall Obligatori
  • Carrega el Pokémon des del paràmetre URL ?id=25
  • Stats amb barres animades (CSS transition)
  • Cadena evolutiva i moviments en taula
  • Alçada, pes, tipus
🎒 my_pokemons.html — Col·lecció Obligatori
  • Mostra Pokémon capturats (persistits al localStorage)
  • Permet ordenar per nom, nivell i número
  • Persisteix entre sessions del navegador
🌿 hunt.html — Caça Obligatori
  • Clic a la zona genera un Pokémon salvatge aleatori
  • Mostra silueta fosca fins que es revela
  • Botó "Lluitar" navega a battle.html via sessionStorage
⚔️ battle.html — Batalla Obligatori
  • Combat per torns amb lògica de dany basada en stats
  • Barres de HP animades, log de combat
  • Pantalles de victòria/derrota
  • En victòria guarda el Pokémon al localStorage

Dissenys

Requisits tècnics de JavaScript

  • Programació modular: cada pàgina té el seu fitxer JS; la lògica compartida (API, storage, model) va a mòduls independents (api.js, pokemon.js, storage.js…).
  • OOP: classe Pokemon amb propietats, mètodes i constructor. La lògica de combat ha d'estar encapsulada en una classe Battle o mòdul equivalent. Valora usar herència o composició si té sentit.
  • Try/Catch: Han de ser explícits. Els errors de xarxa han de mostrar feedback a la UI.
  • Manipulació del DOM: construeix elements amb createElement i textContent; evita innerHTML amb strings no sanititzats.
  • Funcions d'ordre superior: utilitza map, filter, find, reduce per tractar les dades de l'API.
  • Gestió d'estat: localStorage per a la col·lecció; sessionStorage per transferir el Pokémon salvatge a la batalla.
🚫 Queda prohibit Frameworks JS (React, Vue…), llibreries de tercers no autoritzades (jQuery, Lodash…), o IA generativa per escriure el codi. Pots usar la documentació de MDN, PokéAPI, i els apunts del mòdul.
